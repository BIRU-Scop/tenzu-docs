### SSO

To provide Single Sign-On features, Tenzu integrate
with [django-allauth](https://docs.allauth.org/en/latest/socialaccount/)
and [django-auth-ldap](https://django-auth-ldap.readthedocs.io/).

- `django-allauth` enable you to configure account management using any
  supported [providers](https://docs.allauth.org/en/latest/socialaccount/providers/index.html),
  including any provider compatible with a supported generic protocols: OAuth 2.0, OpenID, SAML, ...
  Each configured provider will appear with a dedicated login button to your users.

- `django-auth-ldap` enable you to use a LDAP directory (like an openLDAP or ActiveDirectory) as source of truth for
  your
  users.
  If configured, the classic authentication with password will call on your LDAP server instead.

Here's one possible configuration for an openLDAP:

```bash
TENZU_LDAP__ACTIVATION='lax'
TENZU_LDAP__SERVER_URI='ldap://tenzu-ldap'
TENZU_LDAP__CONNECTION_OPTIONS='{"8": 0}'
TENZU_LDAP__BIND_AS_AUTHENTICATING_USER='False'
TENZU_LDAP__BIND_DN="cn=readonly,dc=tenzu,dc=dev"
TENZU_LDAP__BIND_PASSWORD="ldap_pwd"
TENZU_LDAP__USER_SEARCH='[{"base_dn": "ou=users,dc=tenzu,dc=dev", "scope": 2, "filterstr": "(|(mail=%(user)s)(uid=%(user)s))"}]'
TENZU_LDAP__GROUP_SEARCH='{"base_dn": "ou=groups,dc=tenzu,dc=dev", "scope": 2, "filterstr": "(objectClass=groupOfUniqueNames)"}'
TENZU_LDAP__GROUP_TYPE='{"class_name": "django_auth_ldap.config.NestedGroupOfUniqueNamesType"}'
TENZU_LDAP__REQUIRE_GROUP='cn=users,ou=groups,dc=tenzu,dc=dev'
TENZU_LDAP__USER_ATTR_MAP='{
    "full_name": "cn",
    "email": "mail",
    "username": "uid"
}'
TENZU_LDAP__USER_QUERY_FIELD="username"
TENZU_LDAP__USER_FLAGS_BY_GROUP='{
    "is_active": "cn=users,ou=groups,dc=tenzu,dc=dev"
}'
```

## Configure Tenzu Frontend

The configuration must be set in a file mounted at this location of the frontend image:
`/usr/share/caddy/assets/configs/config.json`.

It is a JSON object expecting the following:
